1 cfg_manager {#cfg_manager}
2 ===========
3 
4 The `cfg_manager` provides a module to load, store, access, and save
5 configuration properties.
6 
7 
8 ## Using the `cfg_manager`
9 
10 The `cfg_manager` is generally loaded via the [`lib_manager`](@ref lib_manager) by:
11 
12  mars::cfg_manager::CFGManagerInterface *cfg;
13  cfg = libManager->getLibraryAs<mars::cfg_manager::CFGManagerInterface>("cfg_manager");
14  if(!cfg) { /* error handling */ }
15 
16 You can access data via the `getOrCreateProperty` method:
17 
18  mars::cfg_manager::cfgPropertyStruct data;
19  data = cfg->getOrCreateProperty(__groupName__, __paramName__,
20  __defaultValue__, NULL);
21  /* acess data.iValue if value type is int */
22 
23 If you inherit from `mars::cfg_manager::CFGClient` and implement the
24 `void cfgUpdateProperty(cfgPropertyStruct _propertyS)` method you can change
25 the last parameter from `NULL` to `this` and the `cfgUpdateProperty` method
26 will be called if the property is changed. Each property gets an unique id
27 which you can use to identify the properties if you register to more than
28 one property:
29 
30  void cfgUpdateProperty(cfgPropertyStruct _propertyS) {
31  if(data.paramId == _propertyS.paramId) {
32  data = _propertyS;
33  }
34  }
35 
36 You can load a configuration file via `cfg->loadConfig(filename)` and save
37 a configuration file for a parameter group via `cfg->writeConfig(filename,
38 groupName)`.
39 
40 ## File Format
41 
42 The configuration files are stored in [yaml](http://www.yaml.org) format with
43 the following syntax:
44 
45  __groupName__:
46  - { name: __paramName__, type: __type__, value: __value__ }
47 
48 or:
49 
50  __groupName__:
51  - name: __paramName__
52  type: __type__
53  value: __value__
54 
55 ## Example
56 
57 c++ access of int "myGroup/myParam":
58 
59  cfg->getOrCreateProperty("myGroup", "myParam", (int)10, this);
60 
61 yaml example:
62 
63  myGroup:
64  - { name: myParam, type: int, value: 10 }
